## Key Design Decisions

1. **`getNotifications` is user-scoped**: Always filters by `userId` as the first `WHERE` condition. No cross-user data leakage.
2. **`getUnreadCount` runs 3 parallel queries**: total unread + urgent count + high count — feeds the bell badge's color logic (`urgent` = red, others = blue).
3. **`NotificationBell` uses SWR-style polling**: `setInterval(POLL_INTERVAL_MS)` inside `useEffect` refreshes the bell counter without a full page reload.
4. **`markAllAsRead` accepts `Record<string, never>`**: Satisfies `createAction`'s `TInput` type parameter while conveying there are no required inputs.
5. **Hard delete for `deleteNotification`**: Notifications have no business value after deletion; no soft-delete needed (no `deletedAt` column in schema).
6. **No slug**: Notifications have no human-readable slug; `getNotificationById` is the only lookup needed.
