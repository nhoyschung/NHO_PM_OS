## 1. 시스템 개요

### 1.1 ProjectOpsOS란?

ProjectOpsOS는 베트남 건설 프로젝트 운영 관리를 위한 웹 기반 시스템입니다. 프로젝트 라이프사이클 전체를 10단계 상태 머신으로 관리하고, 핸드오버 추적, 문서 관리, 태스크 관리, 재무 기록, 규정 준수 모니터링, 감사 로그 등 10개 핵심 모듈을 제공합니다.

시스템 UI는 **베트남어(Tiếng Việt)**로 제공됩니다. 이 매뉴얼에서는 실제 화면에 표시되는 베트남어 레이블을 괄호 안에 병기합니다.

### 1.2 대상 사용자

- 건설 프로젝트 관리자 및 임원
- 프로젝트 팀 리더 및 팀원
- 재무 담당자
- 외부 파트너 (제한된 접근 권한)
- 시스템 관리자

### 1.3 시스템 요구사항

| 항목 | 요구사항 |
|------|---------|
| 브라우저 | Chrome 90+, Firefox 90+, Safari 15+, Edge 90+ |
| 화면 해상도 | 최소 1280x720 (데스크톱), 모바일 반응형 지원 |
| 네트워크 | 인터넷 연결 필수 |
| 쿠키/JS | JavaScript 활성화, 쿠키 허용 필수 |

### 1.4 아키텍처 개요

```
┌─────────────┐
│   Nginx     │  :80/:443  (TLS 종단, 속도 제한)
└──────┬──────┘
       │
┌──────▼──────┐
│  Next.js 15 │  :3000  (App Router + Server Actions + Auth.js RBAC)
└──────┬──────┘
       │
┌──────▼──────┐
│ PostgreSQL  │  :5432  (Drizzle ORM)
└─────────────┘
```

- **프론트엔드/백엔드**: Next.js 15 (App Router, Server Actions)
- **데이터베이스**: PostgreSQL 16 + Drizzle ORM
- **인증**: Auth.js (NextAuth v5) + JWT + bcrypt 비밀번호 해싱
- **UI 프레임워크**: Tailwind CSS 4 + shadcn/ui + Lucide 아이콘
- **유효성 검사**: 모든 경계에서 Zod 스키마 적용
- **역방향 프록시**: Nginx 1.27 (TLS, gzip, 속도 제한 30r/s)

## 2. 로그인 및 계정

### 2.1 로그인 방법

1. 브라우저에서 시스템 URL에 접속합니다 (개발 환경: `http://localhost:3000`).
2. 로그인 페이지 (`/login`)가 표시됩니다.
3. 이메일 주소와 비밀번호를 입력합니다.
4. **"Đăng nhập"** (로그인) 버튼을 클릭합니다.
5. 인증 성공 시 대시보드 (Tổng quan)로 이동됩니다.

## 3. 대시보드 (Tổng quan)

**경로**: `/dashboard`

대시보드는 시스템 접속 후 가장 먼저 표시되는 화면으로, 조직의 프로젝트 포트폴리오에 대한 전체적인 개요를 제공합니다.

### 3.1 화면 구성

대시보드는 크게 세 영역으로 구성됩니다:

1. **통계 카드 영역** (Thẻ thống kê) -- 화면 상단
2. **차트 영역** -- 프로젝트 단계별 분포, 태스크 현황
3. **최근 활동 피드** (Hoạt động gần đây) -- 화면 하단

### 3.2 통계 카드 (Thẻ thống kê)

6개의 핵심 지표 카드가 3열 그리드로 표시됩니다.

| 카드 | 베트남어 레이블 | 의미 | 색상 규칙 |
|------|---------------|------|----------|
| 총 프로젝트 | Tổng dự án | 시스템 내 전체 프로젝트 수, 하위에 활동 중인 프로젝트 수 표시 | 파란색 |
| 진행 중인 프로젝트 | Dự án đang thực hiện | `in_progress` 단계의 프로젝트 수, 하위에 `review` 단계 수 표시 | 보라색 |
| 지연 태스크 | Công việc quá hạn | 마감일이 지난 미완료 태스크 수 | 1개 이상: 빨간색, 0개: 초록색 |
| 대기 중 핸드오버 | Bàn giao chờ phê duyệt | 승인 대기 중인 핸드오버 수 | 1개 이상: 주황색, 0개: 초록색 |
| 재무 잔액 | Số dư tài chính | 총 수입 - 총 지출, 하위에 수입/지출 요약 | 양수: 초록색, 음수: 빨간색 |
| 완료된 태스크 | Công việc hoàn thành | `done` 상태의 태스크 수, 하위에 진행 중인 태스크 수 | 초록색 |

### 3.3 프로젝트 단계 차트 읽는 법

프로젝트 단계 차트(Biểu đồ giai đoạn)는 전체 프로젝트가 10개 단계 중 어디에 분포하는지를 시각적으로 보여줍니다. 각 단계별 색상은 4.4절 "단계 전환" 표를 참조하십시오.

### 3.4 태스크 현황 읽는 법

태스크 현황(Tổng quan công việc)은 전체 태스크를 상태별(열린, 진행 중, 완료, 지연)로 집계하여 표시합니다. 마감일이 지난 미완료 태스크는 "지연(Quá hạn)"으로 별도 강조됩니다.

### 3.5 최근 활동 피드 (Hoạt động gần đây)

시스템 전체에서 발생한 최근 활동을 타임라인 형태로 표시합니다.

- 각 항목에는 **행위 배지** (Tạo mới, Cập nhật, Xóa 등)가 색상으로 구분되어 표시됩니다.
- **엔티티 유형** (Dự án, Công việc, Bàn giao 등)과 이름이 함께 표시됩니다.
- 클릭 가능한 엔티티는 해당 상세 페이지로 이동할 수 있습니다.
- 시간은 상대 시간으로 표시됩니다 ("Vừa xong" = 방금, "5 phút trước" = 5분 전, "2 giờ trước" = 2시간 전).
- **"Xem tất cả"** (모두 보기) 링크를 클릭하면 전체 감사 로그 페이지로 이동합니다.

## 4. 프로젝트 관리 (Dự án)

**경로**: `/dashboard/projects`

프로젝트 모듈은 ProjectOpsOS의 핵심 모듈로, 10단계 상태 머신 기반의 프로젝트 라이프사이클 관리를 제공합니다.

### 4.1 프로젝트 목록 (Danh sách dự án)

#### 화면 구성

- **제목**: "Danh sách dự án" (프로젝트 목록)
- **부제**: "Quản lý tất cả các dự án trong hệ thống" (시스템 내 모든 프로젝트 관리)
- **생성 버튼**: 오른쪽 상단의 **"Tạo dự án mới"** (새 프로젝트 생성) -- admin, manager 역할만 표시

#### 검색 및 필터링

필터 바에는 4가지 필터가 제공됩니다:

| 필터 | UI 레이블 | 설명 |
|------|----------|------|
| 키워드 검색 | "Tìm kiếm dự án..." | 프로젝트 이름으로 자유 텍스트 검색 |
| 단계 필터 | "Giai đoạn" | 10개 단계 중 선택 |
| 우선순위 필터 | "Ưu tiên" | Nghiêm trọng/Cao/Trung bình/Thấp 중 선택 |
| 상태 필터 | "Tình trạng" | Đúng tiến độ/Có rủi ro/Trễ tiến độ/Bị chặn 중 선택 |

필터를 변경하면 자동으로 1페이지로 돌아갑니다. 필터 값은 URL 파라미터에 저장되므로 URL을 공유하여 필터된 뷰를 다른 사람과 공유할 수 있습니다.

#### 테이블 컬럼

| 컬럼 | 베트남어 헤더 | 설명 |
|------|-------------|------|
| 프로젝트명 | Tên dự án | 프로젝트 이름 + 지역(성/시) |
| 프로젝트 코드 | Mã dự án | PRJ-001 형식의 고유 코드 |
| 단계 | Giai đoạn | 현재 프로젝트 단계 (색상 배지) |
| 우선순위 | Ưu tiên | 우선순위 레벨 (색상 배지) |
| 담당자 | Người phụ trách | 프로젝트 매니저 이름 |
| 생성일 | Ngày tạo | 프로젝트 생성 날짜 |

테이블의 행을 클릭하면 해당 프로젝트의 상세 페이지로 이동합니다.

#### 페이지네이션

- 기본 페이지당 표시 건수: **20건**
- 최대 페이지당 표시 건수: **100건**
- 결과 표시: "Hiển thị 1-20 trên 45 dự án" (1-20/45 프로젝트 표시)
- 페이지 네비게이션: 첫 페이지, 이전, 페이지 번호, 다음, 마지막 페이지 버튼 제공
- 페이지 수가 7페이지를 초과하면 말줄임표(...)로 축약 표시

#### 빈 상태

프로젝트가 없을 경우 "Chưa có dự án nào" (아직 프로젝트가 없습니다) 메시지와 함께 **"Tạo dự án mới"** 버튼이 표시됩니다.

### 4.2 프로젝트 생성 (Tạo dự án mới)

**경로**: `/dashboard/projects/new`

#### 생성 절차

1. 프로젝트 목록에서 **"Tạo dự án mới"** 버튼을 클릭합니다.
2. 양식이 표시됩니다 (제목: "Tạo dự án mới").
3. 필수 및 선택 필드를 입력합니다.
4. **"Lưu"** (저장) 버튼을 클릭합니다.
5. 유효성 검사 통과 시 프로젝트가 생성되고 상세 페이지로 이동합니다.
6. **"Hủy"** (취소) 버튼으로 목록 페이지로 돌아갈 수 있습니다.

#### 입력 필드 상세

| 필드 | 베트남어 레이블 | 필수 여부 | 유효성 검사 규칙 | 설명 |
|------|---------------|----------|----------------|------|
| 프로젝트명 | Tên dự án | **필수** | 최소 3자, 최대 200자 | 프로젝트의 고유한 이름 |
| 지역 | Tỉnh/Thành phố | 선택 | 63개 성/시 드롭다운 선택 | 베트남 내 프로젝트 위치 |
| 설명 | Mô tả | 선택 | 최대 2,000자 | 프로젝트에 대한 상세 설명 |
| 시작일 | Ngày bắt đầu | 선택 | 날짜 형식 (YYYY-MM-DD) | 프로젝트 시작 예정일 |
| 종료 예정일 | Ngày kết thúc dự kiến | 선택 | 날짜 형식, 시작일 이후여야 함 | 프로젝트 완료 예정일 |
| 예산 | Ngân sách (VND) | 선택 | 0 이상의 정수, 1,000 단위 | VND 단위 프로젝트 예산 |
| 우선순위 | Ưu tiên | 선택 (기본값: Trung bình) | critical/high/medium/low | 프로젝트 우선순위 |
| 카테고리 | Danh mục | 선택 | 최대 100자 | 프로젝트 분류 (예: Xây dựng, CNTT, Giáo dục) |

> **유효성 검사 메시지 (베트남어)**:
> - 이름 누락: "Tên dự án phải có ít nhất 3 ký tự" (프로젝트 이름은 최소 3자 이상이어야 합니다)
> - 예산 음수: "Ngân sách phải >= 0" (예산은 0 이상이어야 합니다)
> - 날짜 역순: "Ngày bắt đầu phải trước hoặc bằng ngày kết thúc dự kiến" (시작일은 종료 예정일보다 이전이거나 같아야 합니다)

#### 통화 및 태그

- **통화**: 기본값 `VND` (3자 고정, 자동 설정)
- **태그**: 최대 20개, 각 태그 최대 50자

### 4.3 프로젝트 상세 (Chi tiết dự án)

**경로**: `/dashboard/projects/{slug}`

프로젝트 상세 페이지는 **2열 레이아웃**으로 구성됩니다.

#### 상단 영역

- **뒤로 가기**: "Quay lại danh sách" (목록으로 돌아가기) 화살표 버튼
- **프로젝트 이름**: 큰 글씨로 표시, 옆에 현재 단계 배지
- **프로젝트 코드**: 이름 아래 작은 글씨
- **수정 버튼**: **"Chỉnh sửa"** (편집) -- 수정 권한이 있는 역할만 표시

#### 단계 전환 바

현재 단계와 전환 가능한 다음 단계가 표시됩니다 (4.4절에서 상세 설명).

#### 6개 탭

왼쪽 2/3 영역에 탭 기반 콘텐츠가 표시됩니다.

| 탭 | 베트남어 레이블 | 아이콘 | 내용 |
|----|---------------|-------|------|
| 총괄 | Tổng quan | FileText | 프로젝트 설명, 진행률 바, 단계 이력 |
| 핸드오버 | Bàn giao | ArrowRightLeft | 연결된 핸드오버 목록, 상태/유형/날짜/체크리스트 진행 |
| 문서 | Tài liệu | FileText | 연결된 문서 목록, 상태/유형/버전/작성자 |
| 태스크 | Công việc | CheckSquare | 연결된 태스크 목록, 코드/상태/우선순위/담당자/마감일 |
| 재무 | Tài chính | DollarSign | 예산 요약 카드 3개 + 수입/지출 요약 3개 + 최근 거래 목록 |
| 감사 로그 | Nhật ký | ScrollText | 프로젝트 관련 감사 로그 타임라인 |

각 탭 레이블 옆에 해당 항목 수가 뱃지로 표시됩니다 (예: "Công việc 12").

#### 오른쪽 사이드바 (1/3 영역)

**정보 카드 (Thông tin dự án)**:

| 필드 | 베트남어 레이블 | 설명 |
|------|---------------|------|
| 건강 상태 | Trạng thái | 색상 배지 (Đúng tiến độ/Có rủi ro/Trễ tiến độ/Bị chặn) |
| 우선순위 | Ưu tiên | 색상 배지 (Nghiêm trọng/Cao/Trung bình/Thấp) |
| 담당자 | Người phụ trách | 프로젝트 매니저 이름 |
| 팀 리더 | Trưởng nhóm | 팀 리더 이름 |
| 부서 | Phòng ban | 소속 부서명 |
| 지역 | Tỉnh/Thành phố | 프로젝트 위치 (성/시) |
| 시작일 | Ngày bắt đầu | 프로젝트 시작일 |
| 종료 예정일 | Ngày kết thúc dự kiến | 프로젝트 완료 예정일 |
| 예산 | Ngân sách | VND 단위 예산 금액 |
| 생성일 | Ngày tạo | 프로젝트 생성 날짜 |

**멤버 카드 (Thành viên)**:

- 활동 중인 멤버 목록 (최대 8명 표시, 초과 시 "+N thành viên khác" 표시)
- 각 멤버의 이름 첫 글자 아바타, 이름, 역할(owner/lead/member/reviewer/observer) 표시
- **"Quản lý thành viên"** (멤버 관리) 링크로 멤버 관리 페이지 이동

#### 재무 탭 상세

재무 탭은 프로젝트별 재무 현황을 상세히 제공합니다:

**예산 요약 카드 (3열)**:
- **Ngân sách phân bổ** (배정 예산): 프로젝트에 설정된 총 예산
- **Đã chi** (지출): 실제 지출된 금액
- **Còn lại** (잔액): 예산 - 지출, 양수면 초록색, 음수면 빨간색

**예산 사용률 바**: 예산 대비 지출 비율을 프로그레스 바로 표시
- 70% 이하: 초록색
- 70~90%: 노란색
- 90% 초과: 빨간색

**수입/지출 요약 카드 (3열)**:
- **Tổng thu** (총 수입): 초록색
- **Tổng chi** (총 지출): 빨간색
- **Số dư** (잔액): 양수 초록색, 음수 빨간색

### 4.4 단계 전환 (Chuyển giai đoạn) -- 상태 머신

ProjectOpsOS는 **10단계** 프로젝트 라이프사이클 상태 머신을 사용합니다. 총 **15개 전환** (전진 9개 + 후퇴 6개)이 정의되어 있습니다.

#### 10단계 전체 설명

| # | 단계 | 베트남어 레이블 | 설명 | 배지 색상 |
|---|------|---------------|------|----------|
| 1 | Initiation | Khởi tạo | 프로젝트 설정, 이해관계자 식별 | 파란색 |
| 2 | Planning | Lập kế hoạch | 세부 계획 수립, 자원 배분 | 보라색 |
| 3 | In Progress | Đang thực hiện | 개발 및 실행 진행 중 | 노란색 |
| 4 | Review | Đánh giá | 코드 검토, 품질 평가 | 주황색 |
| 5 | Testing | Kiểm thử | 기능 테스트, 통합 테스트 | 청록색 |
| 6 | Staging | Tiền triển khai | 사전 배포 환경 검증 | 남색 |
| 7 | Deployment | Triển khai | 프로덕션 환경 배포 | 에메랄드색 |
| 8 | Monitoring | Giám sát | 배포 후 모니터링 | 틸색 |
| 9 | Handover | Bàn giao | 프로젝트 인수인계, 지식 이전 | 호박색 |
| 10 | Completed | Hoàn thành | 프로젝트 완료, 보관 | 초록색 |

#### 허용된 전환 (15개)

| 현재 단계 | 전환 가능한 단계 | 방향 | 트리거 | 필요 역할 | 조건 |
|-----------|----------------|------|-------|----------|------|
| Khởi tạo | Lập kế hoạch | 전진 | approve_charter | admin, manager, lead | 소유자와 최소 1명의 팀원 필요 |
| Lập kế hoạch | Đang thực hiện | 전진 | approve_plan | admin, manager, lead | 시작일, 종료 예정일, 최소 1개 태스크 필요 |
| Đang thực hiện | Đánh giá | 전진 | submit_for_review | admin, manager, lead, member | 모든 critical/high 우선순위 태스크 완료 또는 검토 중 |
| Đánh giá | Kiểm thử | 전진 | approve_review | admin, manager, lead | 검토 체크리스트 완료 |
| Đánh giá | Đang thực hiện | 후퇴 | request_changes | admin, manager, lead | 검토 의견 문서화됨 |
| Kiểm thử | Tiền triển khai | 전진 | tests_passed | admin, manager, lead | 모든 테스트 통과, 크리티컬 버그 없음 |
| Kiểm thử | Đang thực hiện | 후퇴 | tests_failed | admin, manager, lead | 실패 테스트를 태스크로 문서화 |
| Tiền triển khai | Triển khai | 전진 | staging_approved | admin, manager | 스테이징 환경 검증 완료 |
| Tiền triển khai | Đang thực hiện | 후퇴 | staging_issues | admin, manager, lead | 스테이징 이슈를 태스크로 문서화 |
| Triển khai | Giám sát | 전진 | deployment_complete | admin, manager, lead | 배포 성공, 헬스 체크 통과 |
| Triển khai | Tiền triển khai | 후퇴 | deployment_rollback | admin, manager | 롤백 사유 문서화 |
| Giám sát | Bàn giao | 전진 | monitoring_complete | admin, manager | 모니터링 기간 경과(기본 48h), 크리티컬 이슈 없음, **핸드오버 필수** |
| Giám sát | Đang thực hiện | 후퇴 | issues_detected | admin, manager | 프로덕션 이슈를 태스크로 문서화 |
| Bàn giao | Hoàn thành | 전진 | handover_accepted | admin, manager | 모든 핸드오버 체크리스트 완료, 수신자 확인 |
| Bàn giao | Giám sát | 후퇴 | handover_rejected | admin, manager | 거절 사유 문서화 |

> **"Hoàn thành" (완료)** 단계에서는 더 이상 전환할 수 없습니다 (종료 상태).

#### 전환 UI 조작 방법

1. 프로젝트 상세 페이지에서 **단계 전환 바** (Stage Transition Bar)를 확인합니다.
2. 현재 단계가 색상 배지로 표시되고, 화살표 오른쪽에 전환 가능한 단계 버튼들이 나열됩니다.
3. 원하는 대상 단계 버튼을 클릭합니다.
4. **확인 다이얼로그** (노란색 배경)가 나타납니다:
   - "Bạn có chắc chắn muốn chuyển sang giai đoạn **[단계명]**?" (정말로 [단계명]으로 전환하시겠습니까?)
   - 선택적으로 **"Ghi chú (tùy chọn)..."** (메모) 입력란에 전환 사유를 기록할 수 있습니다 (최대 1,000자).
5. **"Xác nhận"** (확인) 버튼으로 전환을 실행하거나, **"Hủy"** (취소) 버튼으로 취소합니다.
6. 전환 실패 시 오류 메시지가 빨간색으로 표시됩니다.

### 4.5 프로젝트 팀 관리

**경로**: `/dashboard/projects/{slug}/members`

프로젝트 상세 페이지의 오른쪽 사이드바에서 **"Quản lý thành viên"** (멤버 관리) 링크를 클릭하여 접근합니다.

#### 프로젝트 멤버 역할

| 역할 | 설명 |
|------|------|
| owner | 프로젝트 소유자 (최고 권한) |
| lead | 프로젝트 리더 |
| member | 일반 팀 멤버 |
| reviewer | 검토자 |
| observer | 관찰자 (읽기 전용) |

#### 멤버 관리 권한

- 멤버 추가/제거는 `project:member:manage` 권한이 필요합니다 (admin, manager, lead).
- 각 멤버의 활성 상태(`isActive`)를 관리할 수 있습니다.

### 5.3 체크리스트 관리

핸드오버 상세 페이지의 **"Kiểm tra"** (체크리스트) 탭에서 체크리스트 항목을 관리합니다.

#### 체크리스트 항목 구성

| 필드 | 설명 | 유효성 검사 |
|------|------|-----------|
| 제목 (title) | 체크리스트 항목 제목 | 최소 1자, 최대 300자 |
| 설명 (description) | 상세 설명 | 최대 1,000자 |
| 카테고리 (category) | 분류 | 7가지 중 선택 (기본: Khác) |
| 우선순위 (priority) | 중요도 | required/recommended/optional (기본: required) |
| 증거 필요 여부 (requiresEvidence) | 증빙 첨부 필요 여부 | true/false |

#### 체크리스트 카테고리

| 카테고리 | 베트남어 레이블 | 설명 |
|---------|---------------|------|
| documentation | Tài liệu | 문서 관련 항목 |
| access_transfer | Chuyển quyền truy cập | 접근 권한 이전 |
| knowledge_transfer | Chuyển giao kiến thức | 지식 전수 |
| tool_setup | Cài đặt công cụ | 도구/환경 설정 |
| review | Đánh giá | 검토 관련 |
| signoff | Ký xác nhận | 서명/확인 |
| other | Khác | 기타 |

#### 체크리스트 우선순위

| 우선순위 | 베트남어 레이블 | 색상 | 설명 |
|---------|---------------|------|------|
| required | Bắt buộc | 빨간색 | 반드시 완료해야 함 |
| recommended | Khuyến nghị | 노란색 | 권장 사항 |
| optional | Tùy chọn | 회색 | 선택 사항 |

#### 체크리스트 사용 방법

1. 각 항목의 체크박스를 클릭하여 완료 상태를 전환합니다.
2. 완료된 항목은 취소선이 표시되고 회색으로 변경됩니다.
3. 완료 시 완료한 사용자 이름과 날짜가 자동 기록됩니다.
4. **"Yêu cầu bằng chứng"** (증거 필요) 표시가 있는 항목은 증빙 자료 첨부가 필요합니다.
5. 상태가 "Hoàn thành" (완료) 또는 "Đã hủy" (취소)인 경우 체크박스가 비활성화됩니다.

#### 진행률 표시

체크리스트 상단에 진행률 바가 표시됩니다: "Tiến độ kiểm tra: 3/5 (60%)"

### 5.4 승인 워크플로우

핸드오버는 7단계 상태 머신을 따릅니다.

#### 상태 흐름

| 상태 | 베트남어 레이블 | 색상 | 전환 가능한 상태 |
|------|---------------|------|----------------|
| draft | Bản nháp | 회색 | pending_review, cancelled |
| pending_review | Chờ đánh giá | 노란색 | in_review, cancelled |
| in_review | Đang đánh giá | 파란색 | approved, rejected |
| approved | Đã duyệt | 초록색 | completed |
| rejected | Bị từ chối | 빨간색 | draft (수정 후 재제출) |
| completed | Hoàn thành | 에메랄드색 | (종료 상태) |
| cancelled | Đã hủy | 회색 | (종료 상태) |

#### 상태별 가용 액션

| 현재 상태 | 버튼 | 베트남어 레이블 | 필요 역할 |
|-----------|------|---------------|----------|
| draft | 검토 제출 | **"Gửi đánh giá"** | admin, manager, lead |
| draft | 수정 | **"Chỉnh sửa"** | admin, manager, lead |
| in_review | 승인 | **"Phê duyệt"** (초록색 버튼) | admin, manager |
| in_review | 거절 | **"Từ chối"** (빨간색 버튼) | admin, manager |

#### 거절 프로세스

1. **"Từ chối"** 버튼을 클릭합니다.
2. 거절 다이얼로그 (빨간색 배경)가 나타납니다: "Từ chối bàn giao"
3. **"Lý do từ chối..."** (거절 사유) 텍스트 영역에 사유를 입력합니다 (**필수**, 최대 2,000자).
4. **"Xác nhận từ chối"** (거절 확인) 버튼으로 거절을 실행합니다.
5. 거절된 핸드오버는 `draft` 상태로 돌아가며, 거절 사유가 빨간색 배경으로 상세 페이지에 표시됩니다.

#### 핸드오버 상세 페이지 구성

**3개 탭**:
- **Kiểm tra** (체크리스트): 체크리스트 항목 관리
- **Tài liệu** (문서): 첨부 문서 목록
- **Chi tiết** (상세): 설명, 비고, 단계 컨텍스트

**사이드바**:
- **Thông tin bàn giao** (핸드오버 정보): 상태, 유형, 프로젝트, 날짜, 체크리스트 진행
- **Các bên** (관계자): 인도자/인수자/승인자 정보

### 7.2 칸반 보드 뷰 (Bảng Kanban)

칸반 보드는 4개 열로 구성됩니다.

| 열 순서 | 상태 | 베트남어 레이블 | 설명 |
|---------|------|---------------|------|
| 1 | todo | Cần làm | 해야 할 태스크 |
| 2 | in_progress | Đang thực hiện | 진행 중인 태스크 |
| 3 | in_review | Đang đánh giá | 검토 중인 태스크 |
| 4 | done | Hoàn thành | 완료된 태스크 |

#### 칸반 카드 구성

각 태스크 카드에는 다음 정보가 표시됩니다:

- 상단: 우선순위 컬러 바 (critical=빨간색, high=주황색, medium=노란색, low=회색)
- 태스크 코드 (TSK-xxx)
- 태스크 제목 (최대 2줄)
- 우선순위 배지 + 프로젝트 코드 배지
- 하단: 담당자 이름 (없으면 "Chưa giao" = 미배정), 마감일
- 지연 태스크: 빨간색 테두리, 마감일 빨간색 + 경고 아이콘

카드를 클릭하면 태스크 상세 페이지로 이동합니다.

### 7.4 상태 전환

태스크는 7단계 상태 머신을 따릅니다.

| 현재 상태 | 베트남어 레이블 | 전환 가능한 상태 |
|-----------|---------------|----------------|
| backlog | Tồn đọng | todo, cancelled |

| in_progress | Đang thực hiện | in_review, todo, cancelled |
| in_review | Đang đánh giá | testing, in_progress, cancelled |
| testing | Đang kiểm thử | done, in_review, cancelled |
| done | Hoàn thành | (종료 상태) |
| cancelled | Đã hủy | backlog (재활성화 가능) |

### 7.5 지연 태스크 알림

마감일이 지난 미완료 태스크는 다음과 같이 처리됩니다:

- 대시보드: "Công việc quá hạn" (지연 태스크) 통계 카드에 빨간색으로 집계
- 알림: `deadline_overdue` 유형의 자동 알림 생성

1. 재무 페이지에서 CSV 가져오기 영역을 찾습니다 (제목: "Nhập CSV").
2. 필수 컬럼 안내를 확인합니다: 파란색 박스에 필수 컬럼 목록이 표시됩니다.
3. CSV 파일을 **드래그 앤 드롭**하거나, 영역을 **클릭하여 파일을 선택**합니다.
4. 파일 형식 검증:
   - `.csv` 확장자만 허용 (다른 형식: "Chỉ hỗ trợ file CSV.")
   - 필수 컬럼 누락 시: "Thiếu các cột bắt buộc: [누락 컬럼명]"
   - 데이터 행 없음: "File CSV không có dữ liệu."
5. 검증 통과 시 미리보기가 표시됩니다: "[파일명] -- [N] dòng dữ liệu" (N개 데이터 행)
6. **"Nhập [N] bản ghi"** (N개 기록 가져오기) 버튼을 클릭합니다.
7. 가져오기 결과가 3열 카드로 표시됩니다:
   - **Thành công** (성공): 초록색 -- 정상 가져오기 건수
   - **Bỏ qua** (건너뛰기): 노란색 -- 중복 등으로 건너뛴 건수
   - **Lỗi** (오류): 빨간색 -- 오류 건수
8. 오류가 있으면 "Chi tiết lỗi:" (오류 상세)에서 행별 오류 메시지를 확인합니다.
9. **"Đặt lại"** (초기화) 버튼으로 다시 시작할 수 있습니다.

### 8.4 승인 워크플로우

재무 기록은 4단계 상태를 거칩니다:

| 상태 | 베트남어 레이블 | 색상 | 설명 |
|------|---------------|------|------|
| pending | Chờ duyệt | 노란색 | 승인 대기 중 |
| approved | Đã duyệt | 초록색 | 승인됨 |
| rejected | Từ chối | 빨간색 | 거절됨 |
| processed | Đã xử lý | 파란색 | 처리 완료 |

승인/거절은 `finance:approve` 권한이 필요합니다 (admin, manager).

### 8.5 재무 요약 카드 (Thẻ tổng hợp)

4개의 요약 카드가 상단에 표시됩니다:

## 10. 감사 로그 (Nhật ký)

**경로**: `/dashboard/audit-logs`

> **접근 권한**: 사이드바에서 "Nhật ký" 메뉴는 admin, manager 역할에만 표시됩니다. lead 역할은 `audit_log:read` 권한은 있지만 사이드바 네비게이션에서는 보이지 않습니다.

감사 로그는 시스템 내 모든 행위를 불변(immutable)으로 기록하여 규정 준수 및 책임 추적을 지원합니다.

### 10.1 타임라인 보기

감사 로그는 시간 역순으로 타임라인 형태로 표시됩니다.

각 항목에는 다음 정보가 포함됩니다:
- **행위 배지**: 색상으로 구분된 행위 유형
- **심각도 배지**: info가 아닌 경우에만 표시
- **엔티티 이름**: 클릭 가능한 경우 해당 상세 페이지로 이동
- **설명**: 행위에 대한 추가 설명
- **사용자**: 행위를 수행한 사용자 이메일
- **시간**: 행위 발생 시간

### 10.2 필터링

다음 기준으로 감사 로그를 필터링할 수 있습니다:
- **행위 유형** (Action): 21개 행위 중 선택
- **엔티티 유형** (Entity type): 10개 유형 중 선택
- **사용자** (User): 특정 사용자로 필터
- **날짜 범위** (Date range): 시작일/종료일 지정
- **심각도** (Severity): 정보/경고/심각

#### 필터 프리셋

| 프리셋 | 베트남어 레이블 | 필터 조건 |
|--------|---------------|----------|
| 전체 | Tất cả | 필터 없음 |
| 심각 | Nghiêm trọng | severity=critical |
| 경고 | Cảnh báo | severity=warning |
| 인증 | Xác thực | action=login |
| 데이터 변경 | Thay đổi dữ liệu | action=create |
| 보안 | Bảo mật | action=permission_grant |

### 10.3 변경 내역 확인

각 수정 로그에는 이전 값(old value)과 새 값(new value)의 변경 차이(diff)를 확인할 수 있습니다.

- **"Tải xuống"** (다운로드) 버튼을 클릭하여 파일을 저장합니다.

## 12. 규정 준수 (Tuân thủ)

**경로**: `/dashboard/compliance`

### 12.1 대시보드 해석

규정 준수 대시보드(Bảng tuân thủ)는 조직의 준수 지표를 시각적으로 보여줍니다. 각 검사 항목에는 상태 표시기가 제공됩니다.

| 속성 | 설명 |
|------|------|

| status | 결과 상태: pass(통과) / fail(실패) / warning(경고) |

### 12.3 통과율 (passRate) 의미

전체 검사 항목 중 `pass` 상태인 항목의 비율을 백분율로 나타냅니다. 100%면 모든 검사를 통과한 것이며, 낮을수록 준수 개선이 필요합니다.

관리자는 시스템 내 **모든 기능에 대한 전체 접근 권한**을 가집니다.

**주요 업무**:
1. **사용자 관리**: 사용자 생성, 역할 할당, 권한 관리
2. **전체 프로젝트 감독**: 모든 프로젝트의 단계 전환 승인
3. **재무 관리**: 재무 기록 생성/수정/삭제, CSV 가져오기, 승인/거절
4. **감사 로그 모니터링**: 전체 시스템 활동 감시, 보안 이벤트 확인
5. **보고서 생성**: 전체 유형의 보고서 생성 및 내보내기
6. **핸드오버 승인**: 핸드오버 검토 및 승인/거절
7. **설정 관리**: 시스템 설정 변경

**일일 확인 사항**:
- 대시보드에서 지연 태스크 수 확인
- 대기 중인 핸드오버 승인 처리
- 감사 로그에서 `login_failed` 및 `critical` 심각도 이벤트 확인
- 재무 기록 승인 대기 건 처리

### 14.2 매니저 (manager -- Quản lý) 가이드

매니저는 admin과 거의 동일한 권한을 가지지만, 시스템 설정 변경은 불가합니다.

**주요 업무**:
1. **프로젝트 라이프사이클 관리**: 프로젝트 생성, 수정, 단계 전환, 보관
2. **팀 구성**: 프로젝트 멤버 관리, 태스크 배정
3. **핸드오버 관리**: 핸드오버 생성, 검토, 승인/거절
4. **재무 관리**: 예산 배정, 비용 기록, 승인, CSV 대량 가져오기
5. **보고서 활용**: 주기적 보고서 생성으로 프로젝트 현황 파악
6. **문서 관리**: 문서 승인, 보관 관리

**핵심 워크플로우**:
- 새 프로젝트 생성 -> 팀 구성 -> 태스크 생성 -> 단계별 전환 관리 -> 핸드오버 -> 완료
- 재무: 월별 CSV 가져오기 -> 검토 -> 승인 -> 보고서 생성

### 14.3 리더 (lead -- Trưởng nhóm) 가이드

리더는 팀 운영에 초점을 맞춘 역할입니다.

**가용 기능**:
1. **프로젝트**: 조회, 수정, 단계 전환, 멤버 관리 (생성/삭제는 불가)
2. **핸드오버**: 생성, 조회, 수정, 검토 제출 (승인/거절은 불가)
3. **문서**: 생성, 조회, 수정, 버전 생성 (승인/보관은 불가)
4. **태스크**: 모든 기능 (생성, 수정, 삭제, 상태 전환, 배정, 댓글)
5. **재무**: 조회만 가능
6. **감사 로그**: 조회만 가능
7. **알림**: 모든 기능

**일일 업무**:
- 태스크 칸반 보드에서 팀원 업무 현황 확인
- 지연 태스크 팔로업
- 새 태스크 생성 및 담당자 배정
- 프로젝트 단계 전환 요청

### 14.4 멤버 (member -- Thành viên) 가이드

멤버는 자신의 업무 범위 내에서 작업하는 역할입니다.

**가용 기능**:
1. **프로젝트**: 조회만 가능
2. **핸드오버**: 조회만 가능
3. **문서**: 생성, 조회, 버전 생성 (수정/삭제 불가)
4. **태스크**: 생성, 조회, 수정, 상태 전환, 댓글 (삭제/배정 불가)
5. **알림**: 자신의 알림 조회 및 읽음 처리

**사용 불가 기능**: 재무, 감사 로그

**일일 업무**:
- 자신에게 배정된 태스크 확인 및 상태 업데이트
- 프로젝트 관련 문서 업로드
- 알림 확인

### 14.5 뷰어 (viewer -- Người xem) 가이드

뷰어는 **완전한 읽기 전용** 역할입니다.

**가용 기능**:
- 프로젝트, 핸드오버, 문서, 태스크 **조회만** 가능
- 자신의 알림 조회 및 읽음 처리

**사용 불가 기능**: 생성, 수정, 삭제, 승인, 재무, 감사 로그

## 15. FAQ 및 문제 해결

### 자주 묻는 질문

**Q1: 프로젝트 단계를 건너뛸 수 있나요?**

아니요. ProjectOpsOS의 상태 머신은 정해진 전환 경로만 허용합니다. 예를 들어 "Khởi tạo" (초기화)에서 바로 "Đang thực hiện" (진행 중)으로 건너뛸 수 없으며, 반드시 "Lập kế hoạch" (계획)을 거쳐야 합니다.

**Q2: "Hoàn thành" (완료) 상태의 프로젝트를 다시 열 수 있나요?**

아니요. "Hoàn thành"은 종료 상태로, 이후 어떤 단계로도 전환할 수 없습니다. 단계 전환 바가 표시되지 않습니다.

**Q3: CSV 가져오기에서 오류가 발생하면 전체가 실패하나요?**

아니요. CSV 가져오기는 행별(row-level) 검증을 수행합니다. 유효한 행은 정상적으로 가져오고, 오류가 있는 행만 건너뜁니다. 결과 화면에서 성공/건너뛰기/오류 건수를 확인할 수 있습니다.

**Q4: 필터 설정을 다른 사람과 공유할 수 있나요?**

네. 필터 값은 URL 파라미터에 저장됩니다. 필터가 적용된 상태에서 브라우저 주소창의 URL을 복사하여 공유하면, 동일한 필터 설정으로 페이지를 볼 수 있습니다.

**Q5: 알림이 실시간으로 업데이트되나요?**

알림 벨의 읽지 않은 알림 수는 30초 간격으로 자동 갱신됩니다. 완전한 실시간은 아니지만 주기적으로 업데이트됩니다.

**Q6: 어떤 역할이 사이드바에서 재무와 감사 로그를 볼 수 있나요?**

사이드바의 "Tài chính" (재무)과 "Nhật ký" (감사 로그) 메뉴는 admin과 manager 역할에게만 표시됩니다. lead 역할은 재무 조회 권한과 감사 로그 조회 권한이 있지만 사이드바 메뉴에서는 보이지 않습니다.

**Q7: 베트남의 63개 성/시가 모두 지원되나요?**

네. 프로젝트 생성/수정 시 지역(Tỉnh/Thành phố) 드롭다운에 5개 직할시(Hà Nội, TP. Hồ Chí Minh, Hải Phòng, Đà Nẵng, Cần Thơ)와 58개 성 등 총 63개 행정구역이 제공됩니다.

### 일반적인 오류와 해결 방법

| 오류 메시지 | 원인 | 해결 방법 |
|------------|------|----------|

| "Lý do từ chối là bắt buộc." | 거절 시 사유 미입력 | 거절 사유를 반드시 입력 |

### 부록 B: 키보드 단축키 및 팁

ProjectOpsOS는 현재 전용 키보드 단축키를 제공하지 않지만, 다음 팁을 활용하면 효율적으로 사용할 수 있습니다:

1. **검색 활용**: 목록 페이지에서 검색바를 활용하여 빠르게 레코드를 찾을 수 있습니다.
2. **URL 필터 공유**: 필터가 적용된 URL을 복사하여 동료와 동일한 뷰를 공유할 수 있습니다.
3. **컬럼 헤더 정렬**: 테이블 컬럼 헤더를 클릭하여 정렬할 수 있습니다.
4. **알림 벨 확인**: 상단 네비게이션의 벨 아이콘으로 읽지 않은 알림 수를 빠르게 확인합니다.
5. **모바일 사용**: 반응형 레이아웃을 지원하므로 모바일 브라우저에서도 사용 가능합니다. 모바일에서는 사이드바가 오버레이 형태로 표시되며, **"Đóng menu"** (메뉴 닫기) 버튼 또는 배경 클릭으로 닫을 수 있습니다.
6. **빵부스러기(Breadcrumb)**: 상세 페이지에서 빵부스러기 내비게이션을 활용하여 상위 목록으로 빠르게 돌아갈 수 있습니다.
